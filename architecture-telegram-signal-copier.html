Telegram Signal Copier — System Architecture

Event-Driven Architecture · 4-Layer Design · MT5 + TradingView Dual Execution Path

📥 Input Layer

  • • Telethon Client — real-time event listener for Telegram channels
  • • TradingView Webhook — PineScript alerts → REST endpoint
  • • Telegram Bot API — manual signal entry via /trade command
  • • OCR Fallback — extract text from signal screenshots

⚙️ Processing Layer

  • • Regex Parser — symbol, action, entry, SL, TP extraction
  • • SQLite DB — raw signal + parsed order + journal storage
  • • Risk Manager — dynamic lot sizing based on balance
  • • Signal Validator — dupe check, price range, blacklist

🚀 Execution Layer (Dual Path)

  • • Path A: MT5 Python API — direct order_send() to broker
  • • Path B: TV Webhook — Flask server → TradingView alerts
  • • Market, Limit, Stop orders — all 6 order types supported
  • • Modify/Close/Partial Close — order lifecycle management

📊 Reporting Layer

  • • Daily Journal — PnL, win rate, trades count, drawdown
  • • Weekly Journal — performance trends, equity curve
  • • Auto-Forward — reports sent to Telegram owner
  • • CSV Export — full trade history for external analysis
